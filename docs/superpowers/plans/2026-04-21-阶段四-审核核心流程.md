# 阶段四：审核核心流程 实现计划

**Goal:** 实现文档审核的核心流程，包括审核任务管理、逐条审核分析、结果生成与汇总。

**Architecture:** 在阶段二、三基础上，构建审核服务层，调用知识库检索和 LLM 分析能力。

**Constraint:** 本阶段不依赖 Dify（推迟到后续），直接使用 Ollama API 进行 LLM 分析。

---

## 核心功能

| 功能 | 描述 |
|------|------|
| **审核任务管理** | 创建、查询、取消审核任务 |
| **逐条审核** | 对文档每个技术条款进行合规性、完整性、一致性审核 |
| **知识库检索** | 使用 PageIndex 检索相关标准条文 |
| **对比分析** | LLM 对比条款与标准，判定问题 |
| **结果生成** | 汇总问题，生成结构化审核报告 |

---

## 数据模型

### 审核任务 (AuditTask)

```python
class AuditTask(BaseModel):
    id: str
    document_id: str  # 待审核文档 ID
    kb_ids: list[str]  # 引用的知识库 ID 列表
    audit_types: list[AuditType]  # compliance/completeness/consistency
    status: Literal["pending", "processing", "completed", "failed", "cancelled"]
    progress: float  # 0.0 ~ 1.0
    started_at: datetime | None
    completed_at: datetime | None
    result: AuditResult | None
    error_message: str | None
```

### 审核问题 (AuditIssue)

```python
class AuditIssue(BaseModel):
    id: int
    type: AuditType  # compliance/completeness/consistency
    location: IssueLocation  # 位置信息
    description: str
    severity: Literal["high", "medium", "low"]
    standard_reference: StandardRef | None
    suggestion: str | None
```

### 审核结果 (AuditResult)

```python
class AuditResult(BaseModel):
    task_id: str
    summary: ResultSummary
    issues: list[AuditIssue]
    raw_analysis: str | None  # LLM 原始输出
```
