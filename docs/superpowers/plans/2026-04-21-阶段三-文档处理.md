# 阶段三：文档处理模块 实现计划

**Goal:** 实现待审核文档的上传、解析、结构识别和临时索引生成。为阶段四的审核流程提供文档处理能力。

**Architecture:** 在阶段二基础上扩展，增加待审核文档模型、文档解析增强、结构识别服务。

**Constraint:** 复用阶段二已搭建的存储层和服务架构，不引入新的外部依赖。

---

## 核心功能

| 功能 | 描述 |
|------|------|
| **待审核文档管理** | 与知识库文档分离的待审核文档 CRUD |
| **文档解析增强** | 提取文本、页数、章节结构 |
| **结构识别** | LLM 辅助识别章节标题、条款编号 |
| **临时索引** | 为待审核文档生成 PageIndex 临时索引 |

## 数据模型

### 待审核文档 (AuditDocument)

```python
class AuditDocument(BaseModel):
    id: str
    name: str
    original_name: str
    file_type: Literal["pdf", "doc", "docx"]
    file_path: str
    page_count: int | None
    status: Literal["uploaded", "parsed", "indexed", "audit_pending", "auditing", "completed", "failed"]
    created_at: datetime
    updated_at: datetime
    parsed_content: str | None  # 提取的纯文本
    structure: dict | None  # 识别出的结构（章节、条款）
    tree_index_path: str | None
```

### 文档结构 (DocumentStructure)

```python
class DocumentStructure(BaseModel):
    title: str | None
    chapters: list[Chapter] = []
    total_clauses: int = 0


class Chapter(BaseModel):
    number: str | None  # 如 "第一章"
    title: str
    clauses: list[Clause] = []


class Clause(BaseModel):
    number: str  # 如 "1.1", "2.1.1"
    text: str
    page: int | None
```
