# 阶段二：知识库管理模块 实现计划

**Goal:** 实现知识库的创建、分类、文档导入和 PageIndex 索引管理。支持 FastAPI REST API 和 CLI 双入口，采用文件系统 + JSON 元数据存储。

**Architecture:** 分层架构 — Controller (FastAPI/typer CLI) → Service (业务逻辑) → Repository (文件系统 CRUD)。PageIndex 作为外部依赖在 Service 层调用。

**Tech Stack:** Python 3.10+, FastAPI, Uvicorn, Typer, Pydantic, PageIndex, pdfplumber, python-docx

**Constraint:** Dify 集成推迟到阶段四。本阶段不依赖外部数据库或 Docker 编排（除 PageIndex/Ollama 外）。

**Previous Phase Context:** 阶段一已完成 Ollama + Qwen3.5-0.8B 部署、PageIndex 索引/检索验证、PDF/Word 文档解析验证。示例文档 `sample_docs/sample_standard.pdf` 可生成。

---

## 核心决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 存储方式 | 文件系统 + JSON 元数据 | 原型阶段零依赖、易调试、Git 可见；后续阶段无缝迁移 PostgreSQL |
| API 形态 | FastAPI + CLI 双入口 | 共享 Service 层；CLI 用于开发/批量调试，API 供后续前端/Dify 调用 |
| Dify 集成 | 阶段四引入 | 避免被平台细节拖慢；阶段二聚焦知识库独立能力 |
| PageIndex 调用方式 | Python 库直接调用 | 阶段一验证通过；通过 service 封装便于后续切换为 HTTP API 或替换引擎 |

## 数据模型

### 知识库 (KnowledgeBase)

```python
class KnowledgeBase(BaseModel):
    id: str  # ulid 格式
    name: str
    description: str
    category: Literal["national", "industry", "enterprise"]
    created_at: datetime
    updated_at: datetime
    document_ids: list[str] = []
    index_status: Literal["none", "building", "ready", "failed"]
```

### 知识库文档 (KBDocument)

```python
class KBDocument(BaseModel):
    id: str  # ulid 格式
    kb_id: str
    name: str
    original_name: str
    file_type: Literal["pdf", "doc", "docx"]
    file_path: str  # kb_docs/{kb_id}/{id}.{ext}
    tree_index_path: str | None  # kb_index/{kb_id}/{id}_tree.json
    page_count: int | None
    created_at: datetime
    updated_at: datetime
    index_status: Literal["none", "building", "ready", "failed"]
    metadata: dict = {}  # 文件大小、解析出的章节数等
```

### 知识库元数据存储格式

每个知识库一个目录 `data/kb_meta/{kb_id}/`：

```
data/kb_meta/
└── {kb_id}/
    ├── kb.json           # KnowledgeBase 序列化
    ├── {doc_id}.json     # KBDocument 序列化
    └── ...
```

### 索引存储格式

```
data/kb_index/
└── {kb_id}/
    └── {doc_id}_tree.json    # PageIndex 树索引
```

## Task 3: Service 层实现（业务逻辑）

**Files:**
- `services/kb_service.py` — 知识库管理：创建、删除、列表、分类筛选
- `services/doc_service.py` — 文档导入：校验格式、解析文本、保存、更新元数据
- `services/indexing_service.py` — 索引管理：调用 PageIndex 生成树索引、更新状态

- [ ] **Step 1: 实现 kb_service.py**
  - `create_kb(name, description, category) -> KnowledgeBase`
  - `get_kb(kb_id) -> KnowledgeBase`
  - `list_kbs(category=None) -> list[KnowledgeBase]`
  - `delete_kb(kb_id) -> bool`（级联删除文档和索引文件）

- [ ] **Step 2: 实现 doc_service.py**
  - `import_document(kb_id, file) -> KBDocument`
    - 校验文件类型（仅限 pdf/doc/docx）
    - 保存文件到 `data/kb_docs/{kb_id}/`
    - pdfplumber 提取页数等元数据
    - 创建 KBDocument 记录
  - `delete_document(kb_id, doc_id) -> bool`（级联删除索引）

- [ ] **Step 3: 实现 indexing_service.py**
  - `build_index_for_doc(doc: KBDocument, model: str)`
    - 调用 PageIndex Python 接口生成树索引
    - 保存到 `index_repo`
    - 更新 doc.index_status 和 kb.index_status
  - `rebuild_kb_index(kb_id: str, model: str)` — 重建整个知识库所有文档的索引

**异步处理：** 索引构建耗时较长（PageIndex 调用 LLM），建议：
- API 层返回 `{"status": "building", "task_id": "..."}`
- 后台线程执行索引（原型阶段简化为同步阻塞，或 threading）；后续阶段升级为 Celery/RQ
